一：Git是什么？</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　Git是目前世界上最先进的分布式版本控制系统。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　二：SVN与Git的最主要的区别？</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　SVN是集中式版本控制系统，版本库是集中放在中央服务器的，而干活的时候，用的都是自己的电脑，所以首先要从中央服务器哪里得到最新的版本，然后干活，干完后，需要把自己做完的活推送到中央服务器。集中式版本控制系统是必须联网才能工作，如果在局域网还可以，带宽够大，速度够快，如果在互联网下，如果网速慢的话，就纳闷了。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　Git是分布式版本控制系统，那么它就没有中央服务器的，每个人的电脑就是一个完整的版本库，这样，工作的时候就不需要联网了，因为版本都是在自己的电脑上。既然每个人的电脑都有一个完整的版本库，那多个人如何协作呢？比如说自己在电脑上改了文件A，其他人也在电脑上改了文件A，这时，你们两之间只需把各自的修改推送给对方，就可以互相看到对方的修改了。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

　　一：创建版本库。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　什么是版本库？版本库又名仓库，英文名repository,你可以简单的理解一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改，删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻还可以将文件”还原”。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

　　三：理解工作区与暂存区的区别？</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　工作区：就是你在电脑上看到的目录，比如目录下testgit里的文件(.git隐藏目录版本库除外)。或者以后需要再新建的目录文件等等都属于工作区范畴。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　版本库(Repository)：工作区有一个隐藏目录.git,这个不属于工作区，这是版本库。其中版本库里面存了很多东西，其中最重要的就是stage(暂存区)，还有Git为我们自动创建了第一个分支master,以及指向master的一个指针HEAD。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　第一步：是使用 git add 把文件添加进去，实际上就是把文件添加到暂存区。</p>
<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">
　　第二步：使用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支上。</p>

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<p style="margin-top:10px; margin-bottom:10px; padding-top:0px; padding-bottom:0px; color:rgb(85,85,85); font-family:Verdana,Arial,Helvetica,sans-serif; font-size:14px; line-height:23.799999237060547px; background-color:rgb(249,249,249)">

<h3>git常用命令：</h3>

                        
<span style="color:rgb(69,69,69); font-family:Tahoma,Arial,Helvetica,STHeiti; font-size:14px"></span>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<a href="http://gitref.org/index.html" target="_blank" style="text-decoration:underline; color:rgb(63,167,203)">Git</a>&nbsp;是一个很强大的分布式版本控制系统。它不但适用于管理大型开源软件的源代码，管理私人的文档和源代码也有很多优势。</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>Git常用操作命令：</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>1)&nbsp;远程仓库相关命令</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
检出仓库：$ git clone git://github.com/jquery/jquery.git</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
查看远程仓库：$ git remote -v</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
添加远程仓库：$ git remote add [name] [url]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除远程仓库：$ git remote rm [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
修改远程仓库：$ git remote set-url --push&nbsp;[name]&nbsp;[newUrl]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
拉取远程仓库：$ git pull [remoteName] [localBranchName]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
推送远程仓库：$ git push [remoteName] [localBranchName]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
*如果想把本地的某个分支test提交到远程仓库，并作为远程仓库的master分支，或者作为另外一个名叫test的分支，如下：</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
$git push origin test:master &nbsp; &nbsp; &nbsp; &nbsp; //&nbsp;提交本地test分支作为远程的master分支</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
$git push origin test:test &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;//&nbsp;提交本地test分支作为远程的test分支</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>2）分支(branch)操作相关命令</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
查看本地分支：$ git branch</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
查看远程分支：$ git branch -r</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建本地分支：$ git branch [name] ----注意新分支创建后不会自动切换为当前分支</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
切换分支：$ git checkout [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建新分支并立即切换到新分支：$ git checkout -b [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除分支：$ git branch -d [name] ---- -d选项只能删除已经参与了合并的分支，对于未有合并的分支是无法删除的。如果想强制删除一个分支，可以使用-D选项</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
合并分支：$ git merge [name] ----将名称为[name]的分支与当前分支合并</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建远程分支(本地分支push到远程)：$ git push origin [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除远程分支：$ git push origin :heads/[name]&nbsp;或&nbsp;$ gitpush origin :[name]&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
*创建空的分支：(执行命令之前记得先提交你当前分支的修改，否则会被强制删干净没得后悔)</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
$git symbolic-ref HEAD refs/heads/[name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
$rm .git/index</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
$git clean -fdx</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>3）版本(tag)操作相关命令</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
查看版本：$ git tag</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建版本：$ git tag [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除版本：$ git tag -d [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
查看远程版本：$ git tag -r</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建远程版本(本地版本push到远程)：$ git push origin [name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除远程版本：$ git push origin :refs/tags/[name]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
合并远程仓库的tag到本地：$ git pull origin --tags</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
上传本地tag到远程仓库：$ git push origin --tags</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
创建带注释的tag：$ git tag -a [name] -m 'yourMessage'</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>4)&nbsp;子模块(submodule)相关操作命令</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
添加子模块：$ git submodule add [url] [path]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;&nbsp;&nbsp;如：$git submodule add git://github.com/soberh/ui-libs.git src/main/webapp/ui-libs</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
初始化子模块：$ git submodule init &nbsp;----只在首次检出仓库时运行一次就行</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
更新子模块：$ git submodule update ----每次更新或切换分支后都需要运行一下</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
删除子模块：（分4步走哦）</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;1)&nbsp;$ git rm --cached [path]</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;2)&nbsp;编辑“.gitmodules”文件，将子模块的相关配置节点删除掉</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;3)&nbsp;编辑“&nbsp;.git/config”文件，将子模块的相关配置节点删除掉</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;4)&nbsp;手动删除子模块残留的目录</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
&nbsp;</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
<strong>5）忽略一些文件、文件夹不提交</strong></p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
在仓库根目录下创建名称为“.gitignore”的文件，写入不需要的文件夹名或文件，每个元素占一行即可，如</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
target</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
bin</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
*.db</p>
<p style="margin-top:0px; margin-bottom:0px; padding-top:0px; padding-bottom:0px">
</p>

git rm --cached a.a 移除文件(只从暂存区中删除)<br>
git commit -m &quot;remove&quot; 移除文件(从Git中删除)<br>
git rm -f a.a 强行移除修改后文件(从暂存区和工作区中删除)<br>
git diff --cached 或 $ git diff --staged 查看尚未提交的更新<br>
git stash push 将文件给push到一个临时空间中<br>
git stash pop 将文件从临时空间pop下来<br>
